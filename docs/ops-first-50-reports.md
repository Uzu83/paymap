# 初回50報告オペレーション（倫理ガイド）

Paymap（使えるペイ）福岡パイロットで、**捏造のない実報告**を最初の50件まで集める手順。審査・実証で「トラクションゼロ」を脱するための運用プレイブック。

## 原則（必読）

1. **実際に店舗で決済した体験のみ報告する** — 推測・口コミ・チェーン一般論での報告はしない
2. **使えなかった報告も同等に価値がある** — 成功だけを集めない（失敗報告こそ差別化）
3. **同一店舗の連打禁止** — システムは IP+店舗で1時間10回まで（`lib/rateLimit.ts`）
4. **サンプル渋谷店への報告は審査本線に含めない** — 福岡 OSM 実在店が対象
5. **店員への許可** — 撮影・店名掲載が必要なオペは事前に商店街経由で調整

## 目標

| 指標 | 目標 |
|---|---|
| 実報告数 | 50件（worked + failed 合算） |
| ユニーク店舗 | 20店以上 |
| 期間 | パイロット M2（4〜8週目） |
| 未検証店の遷移 | 少なくとも10店が報告付きに |

## 対象ストリート（優先順）

天神・博多エリアの歩行導線に沿って段階的に広げる。

1. **天神地下街〜ソラリアステージ周辺** — 人流多・チェーンと個人店が混在
2. **渡辺通・今泉** — 飲食密度が高く未検証店が多い
3. **博多駅筑紫口〜博多阪急前** — インバウンド導線
4. **中洲（屋台・居酒屋）** — 現金のみ店が残るエリア（失敗報告の価値大）
5. **櫛田神社・川端通商店街** — 商店街連携のテストケース

各エリアは `data/shops.json` の `paymentStatus: "unverified"` 店を優先。チェーン推定店は検証の優先度を下げる。

## レート制限（技術）

- **API**: 同一 IP ハッシュ + 店舗 ID で **1時間に10回まで**
- **運用**: 1人1店1回が基本。複数決済手段は別レシート体験がある場合のみ
- **ボランティア**: 3人×週2回×4週 ≒ 24店訪問が現実的な上限感

## ボランティア向けスクリプト（30秒）

> こんにちは。福岡の飲食店で「PayPay やカードが使えるか」を地図に載せる市民プロジェクトをしています（Paymap / 使えるペイ）。
>
> 今日このお店で実際に使った決済について、地図アプリから「使えた」「使えなかった」を1タップで報告していただけると助かります。店舗名の掲載は OSM 由来の公開情報です。お時間2分いただけますか？

**断られた場合**: 無理に勧めない。自分の購買体験のみ報告する。

**報告手順**:

1. https://paymap-six.vercel.app を開く（福岡エリア）
2. 訪問した店のピンをタップ
3. 実際に使った決済で「今使えた」または「使えなかった」
4. 未検証バッジの店は特に価値が高い

## 週次チェックリスト

- [ ] 新規ユニーク店舗数（前週比）
- [ ] worked / failed 比率（極端な偏りはオペミスを疑う）
- [ ] 同一 IP からの異常集中（なければ OK）
- [ ] 未検証 → 報告ありの遷移数
- [ ] 商店街・観光案内への導線掲載状況

## やってはいけないこと

- 報告数を水増しする（複数アカウント・虚偽体験）
- チェーン一般論だけで「使えた」と報告する
- 閉店・移転店への古い情報の投稿
- 店舗を特定できる写真・個人情報の収集
