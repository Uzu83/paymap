# 決済ステータス昇格ルール（field）

虚偽の「現地確認済」は禁止。以下を満たしたときだけ `paymentStatus: "field"` に上げる。

## 条件（いずれか）

1. **運営者自身の店頭確認**: 日付・確認した決済手段・確認者メモを `note` に残す  
2. **独立した報告が同一手段で成功3以上、かつ失敗0**（または成功≥失敗×3 かつ 合計≥5）  
3. **店舗公式（Web/掲示）のスクリーンショットを運営が確認**（リポジトリに画像は必須ではないが確認日を記録）

## 降格

- 失敗報告が直近で連続した場合は `estimated` または `unverified` に戻す
- サンプル店は常に `sample`（field にしない）

## UI

- `field` → 「決済: 現地確認済」
- 報告0件の空状態では「最初の報告者が精度を作ります」と表示（捏造誘導しない）

## 自動化（将来）

現状は手動シード更新。報告閾値での自動昇格は Round 後続で API 集計から検討。
