**福岡市向け — 店舗キャッシュレス可否マップ**

市民・観光客の「今使えた / 使えなかった」報告で鮮度を上げる Web アプリ（個人開発プロトタイプ）

## 課題

飲食・小売の直前に「PayPay / クレジットが通るか」が分からず、機会損失と現金依存が残る。Google マップ等の決済情報は鮮度が弱く、失敗報告を反映する仕組みがない。福岡市 DX 戦略のキャッシュレス推進に、民間店舗の見える化が足りない。

## ソリューション

天神〜博多の実在店（OSM シード約68件）を地図表示。決済手段でフィルタし、利用者報告で信頼度を更新。チェーン推定・未検証をバッジで明示し、虚偽トラクションは出さない。

## デモ

https://paymap-six.vercel.app — 地図 → ピン → 「今使えた / 使えなかった」

## 3ヶ月パイロット KPI（案）

- 対象エリアの実在店カバー率
- 週次ユニーク報告数（レート制限あり）
- 「決済前に確認した」利用者率
- 未検証店 → 報告付き店への遷移率

## お願い（Ask）

- パイロットエリアの商店街・商工会への紹介
- 観光案内所等への地図導線の試験掲載
- 実証期間中の告知協力
